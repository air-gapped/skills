| NIXL 1P1D bench vs single-pod baseline on consumer lab | live lab verification | 2026-04-25 | `vllm bench serve --random-input-len 512 --random-output-len 64 --num-prompts 30 --request-rate 3`. 1P1D via proxy: 1.65 req/s, 965 tok/s, mean TTFT 3786 ms, TPOT 37.9 ms. Direct to single pod (same HW, no NIXL): 2.21 req/s, 1291 tok/s, mean TTFT 411 ms, TPOT 36.8 ms. 1P1D **−34 % throughput**, TTFT **9× worse**. Reason: short decode-dominated prompts give NIXL no prefill savings to recover, while same-node `cuda_copy,sm,tcp` transfer adds per-request overhead and the prefiller serialises all arrivals. Disaggregation pays off when input ≫ output (16 k+ prefill), prefill latency dominates TTFT, 1P:N-D ratio, AND multi-node where prefill pool scales independently. Consumer single-node short-prompt loads: use a single pod + prefix caching instead. |

## 2026-07-21 freshen pass

vLLM moved four minors (v0.21.0 → v0.25.1) since the last pass and two of the changes invert core skill guidance. (1) **v0.22.0 shipped a native multi-tier offloading framework** — `OffloadingConnector` + `spec_name: TieringOffloadingSpec` with `fs` / `obj` / `p2p` secondary tiers — so an NVMe or object tier no longer requires LMCache; documented upstream in the new `docs/features/kv_offloading_usage.md`. (2) **v0.23.0 made HMA the default for capable connectors (#41847)**, turning `disable_hybrid_kv_cache_manager` into a tri-state auto flag; the skill's "single most common silent blocker — always pass `--disable-hybrid-kv-cache-manager`" pitfall was retired and replaced with a per-connector `SupportsHMA` table verified against v0.25.1 source. Also: `P2pNcclConnector` removed in v0.24.0 (#44854); NixlConnector `kv_both` deprecation cycle begun (#43874); `nixl` pin tightened to `== 1.3.0`. Bug state: vLLM #40624 (Gemma4 + spec-decode) closed COMPLETED 2026-05-26; #40259 still open; LMCache #2942 open-but-stale-flagged; #3106 open and active. LMCache 0.5.x now supports hybrid models in MP mode — cross-referenced to the `lmcache-mp` skill rather than duplicated here.
