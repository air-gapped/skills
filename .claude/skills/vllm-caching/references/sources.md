| NIXL 1P1D disaggregated prefill verified on consumer lab (2× 4060 Ti) | live lab verification | 2026-04-25 | Prefiller pod (GPU 0, `kv_role=kv_producer`) + decoder pod (GPU 1, `kv_role=kv_consumer`) + CPU-only proxy pod running `toy_proxy_server.py`. Qwen3-4B + vLLM v0.19.2rc1. Client POST → proxy → prefiller 2-token prefill → NIXL UCX transfer → decoder 10-token decode → stream back. Round-trip 755 ms. Decoder log: `NIXL compatibility check passed` + `Transfer plan: TransferTopology(tp_ratio=1, K=8, local_tp=1, remote_tp=1, local_rank=0, remote_block_len=65536)`. **Three non-obvious gotchas**: (1) `UCX_TLS=cuda_copy,sm,tcp` is mandatory — `tcp` alone segfaults in `nixlUcxSharedThread::run()` after prefill with `W ucx_utils.cpp:581: memory is detected as host`; image's UCX IS CUDA-capable but TLS defaults don't enable cuda_copy. (2) First request after pod-ready may fail with `NIXL_ERR_REMOTE_DISCONNECT` (handshake race); second+ works. (3) Proxy httpx client pool caches pod IPs — restart proxy if you restart vLLM pods. | NIXL 1P1D bench vs single-pod baseline on consumer lab | live lab verification | 2026-04-25 | `vllm bench serve --random-input-len 512 --random-output-len 64 --num-prompts 30 --request-rate 3`. 1P1D via proxy: 1.65 req/s, 965 tok/s, mean TTFT 3786 ms, TPOT 37.9 ms. Direct to single pod (same HW, no NIXL): 2.21 req/s, 1291 tok/s, mean TTFT 411 ms, TPOT 36.8 ms. 1P1D **−34 % throughput**, TTFT **9× worse**. Reason: short decode-dominated prompts give NIXL no prefill savings to recover, while same-node `cuda_copy,sm,tcp` transfer adds per-request overhead and the prefiller serialises all arrivals. Disaggregation pays off when input ≫ output (16 k+ prefill), prefill latency dominates TTFT, 1P:N-D ratio, AND multi-node where prefill pool scales independently. Consumer single-node short-prompt loads: use a single pod + prefix caching instead. |
